Основа проекта: ASP.NET core, .net 6, PostgreSQL 15.
Дополнительные библиотеки которые были использованы:

1. Microsoft.EntityFrameworkCore - для облегчения работы с БД, на основе ORM.
2. Microsoft.EntityFrameworkCore.Proxies - используется для облегчения работы с EntityFrameworkCore, а именно для ленивой подгрузки свойств навигации у моделей.
3. Newtonsoft.Json - для облегчения работы с сериализацией и десириализацией в json cущностей моделей.
4. Npgsql.EntityFrameworkCore.PostgreSQL - облегчает работу с БД PostgreSQL, позволяет проще создать репозиторий сущностей.
5. System.Configuration.ConfigurationManager - используется для облегчения конфигурации настроек приложения.

Для того чтобы задать строку подключения к базе данных и путь хранения логов воспользуйтесь файлом app.config.

Архитектура приложения: т.к. за основу взят Microsoft.EntityFrameworkCore, то и для реализации взаимосвязей используется построение моделей(папка Models).
Репозиторий сущностей ApplicationContext - связующее звяно между БД и приложением. Там находится конфигурация подключения и сами сущности.
APIManager - отвечает за всю логику поиска/создания/изменения/удаления сущностей репозитория. Для удобства масштабирования системы всё сделано через
рефлексию, что позволяет просто создавать модели и добавлять их в репозиторий(не дорабатывая при этом логику crud операций с новой моделью).
Program - ключевое место приложения, происходят некоторые конфигурации, отвечает за маппинг страниц, обрабатывает Api запросы.

Так же добавлена авторизация пользователя, но на зачаточном уровне, как пример возможной реализации.
